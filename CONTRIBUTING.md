# Contributing to PawMatch

Thank you for considering contributing to PawMatch. Contributions of all kinds are welcome, including bug fixes, new features, documentation improvements, testing, and discussion.

This document provides guidelines to help maintain quality and consistency across the project.

---

## How to Contribute

### 1. Fork the repository

Create your own fork of the project.

### 2. Clone your fork locally

```
git clone https://github.com/<your-username>/pawmatch.git
cd pawmatch
```

### 3. Create a new branch

```
git checkout -b feature/your-feature-name
```

### 4. Make your changes

Keep code clean, readable, and consistent.

### 5. Commit your changes

```
git commit -m "Add: short description of the change"
```

### 6. Push to your fork

```
git push origin feature/your-feature-name
```

### 7. Open a Pull Request

Explain what and why, reference issues if applicable.

---

## Reporting Issues

When reporting a bug or suggestion:

* Include steps to reproduce
* Expected vs actual behavior
* Logs or screenshots when helpful
* Check existing issues first

---

## Commit Message Examples

```
Fix: crash when creating dog profile
Add: AI module for bio generation
Update: improve match ranking
Docs: update setup section
```

---

## Pull Request Guidelines

* Keep changes focused and atomic
* Add tests where possible
* Update documentation if needed

---

## Questions or Discussion

Open an issue with the title `Question: <topic>` if you need help.

Thank you for contributing to PawMatch.
